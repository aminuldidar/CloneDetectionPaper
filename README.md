# Replication Package: Validation of Structural Fidelity in Method-Level Clone Detection

This repository contains the replication data, evaluation corpora, and human oracle validation datasets for our large-scale empirical study of structural code cloning.

---

## 📁 Repository Directory Structure

```text
├── README.md                               <- This documentation file
├── downsized_clone_provenance_dataset.csv   <- Curated global ecosystem clone provenance mapping
├── clone_pair_similarity.csv               <- Deep AST analysis features and raw code metrics
└── human_annotation_oracle.csv             <- Dual-annotated human baseline evaluation corpus


### Dataset Schemas & Data Dictionaries

1. Global Provenance Data (downsized_clone_provenance_dataset.csv)
Contains the foundational structural links across the software ecosystem, optimized for rapid mapping between origin code bases and cloned targets.

Column Name,Type,Description
hash,String,Cryptographic invariant match fingerprint generated from normalized AST subtrees.
source_method_name,String,Original signature identifier of the root method declaration.
sink_method_name,String,Secondary signature identifier of the cloned method target copy.
source_repository_url,String,Target public repository web route hosting the original source logic.
sink_repository_url,String,Destination public repository web route hosting the duplicated logic.
source_file_location,String,Core file resolution path tracking the origin file name and entry line number.
sink_file_location,String,Destination file resolution path tracking the target file name and entry line number.


2. AST Similarity Metrics Data (clone_pair_similarity.csv)
Maintains the dense similarity scores and structural dimensions for evaluation. It includes multi-line text blocks representing the exact code state harvested from the repository histories.

Column Name,Type,Description
id,Integer,Unique clone pair tracking trace identifier.
common_hash,String,Corresponding 128-bit structural invariant AST subtree hash.
sink_url,String,Code link pointing directly to the target/duplicated method line location.
sink_version,BigInt,Unix epoch timestamp representing the target repository capture version.
sink_path,String,Relative local path and line location for the target code block.
sink_method_name,String,Extracted identifier name of the method inside the target repository.
sink_code_db,String,Internal caching registry configuration state flag.
sink_code_live,String,[Raw Code Block] Full multi-line source code block of the target method snippet.
sink_granular_level,Integer,Target node resolution structural granularity level assignment.
source_url,String,Original hosting repository codebase link where the clone trace originated.
source_version,BigInt,Unix epoch timestamp representing the origin repository reference version.
source_path,String,Relative local path and line location for the original source code block.
source_method_name,String,Extracted identifier name of the method inside the origin repository.
source_code_db,String,Internal caching registry configuration state flag.
source_code_live,String,[Raw Code Block] Full multi-line source code block of the original source snippet.
source_granular_level,Integer,Source node resolution structural granularity level assignment.
live_ast_similarity_score,Float,Automated structural similarity score (S) generated by our AST parser engine.
verification_timestamp,String,Timestamp tracking runtime processing execution tracking logs.
sink_line_count,Integer,Total metrics line count of the target method code fragment.
source_line_count,Integer,Total metrics line count of the origin source code fragment.
language,String,"Program parsing syntax targeting domain label (e.g., python, cpp)."

3. Human Validation Oracle (human_annotation_oracle.csv)
Extends the core metric schema to include independent categorical evaluations by two software engineering experts, documenting inter-rater reliability boundaries.

Inherits all attributes from clone_pair_similarity.csv alongside the following explicit validation additions:

Column Name,Type,Description
label_1,Integer,"Classification choice from Annotator 1 (1 = True Clone, 0 = False Positive)."
label_2,Integer,"Classification choice from Annotator 2 (1 = True Clone, 0 = False Positive)."
bucket,String,"Categorized structural grouping boundary assigned by score tiers (e.g., Type-1, Type-2)."
agree,Boolean,Operational flag indicating structural convergence between label_1 and label_2.
